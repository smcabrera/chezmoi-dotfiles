# Git Commit Skill Triggers

This skill should be activated when the user asks to:

## Primary Triggers
- "commit" (as a verb in the context of git/code changes)
- "make a commit"
- "create a commit"
- "git commit"
- "commit these changes"
- "commit the changes"
- "commit my changes"
- "commit this"
- "commit that"

## Secondary Triggers (with context)
- "save these changes" (when in a git repository context)
- "check in" (when referring to code)
- "push" (when it implies committing first)

## Trigger Phrases Examples
- "Can you commit this with the message..."
- "Please make a commit"
- "Commit these changes"
- "I need you to commit this"
- "Create a git commit"
- "Time to commit"
- "Let's commit this work"
- "Commit with message:"
- "Make a commit saying..."

## Context Clues
The skill should also activate when:
- The user provides a commit message in quotes after mentioning changes
- The conversation involves code changes that need to be saved
- The user asks about git status and then implies committing

## Exclusions
Do NOT trigger this skill for:
- "commit" used in other contexts (e.g., "commit to memory", "commit a crime")
- General git questions without intent to actually commit
- Discussions about commit history or past commits
